# Machine Learning Engineering — Data Pipeline

PySpark pipeline for **credit default prediction** at a digital lender. Raw LMS and feature CSVs are ingested, cleaned, and materialized into a **medallion datamart** (bronze → silver → gold). Gold delivers a **label store** (30+ DPD at month 6) and an **ML-ready feature store** (leakage-safe joins, encodings, train-window winsorization and imputation) for downstream modeling.

## Medallion architecture

| Layer | Role | Outputs (under `datamart/`) |
|-------|------|-----------------------------|
| **Bronze** | Snapshot raw CSVs by source and month | `bronze/{clickstream,attributes,financials,loan_daily}/` |
| **Silver** | Type casting, `mob`/`dpd` on loan daily | `silver/{source}/silver_*_YYYY_MM_DD.parquet` |
| **Gold** | Label + feature stores for modeling | `gold/label_store/`, `gold/feature_store/` |

**Sources:** `data/feature_clickstream.csv`, `data/features_attributes.csv`, `data/features_financials.csv`, `data/lms_loan_daily.csv`.

**Gold outputs:**

- **Label store** — `label = 1` if `dpd ≥ 30` at `mob = 6`; `label_def = "30dpd_6mob"`. No feature transforms on the target.
- **Feature store** — One row per `(Customer_ID, snapshot_date)` (~12,500 rows). Application-time attributes and financials, clickstream aggregates up to `loan_start_date`, encodings, missingness flags, age validity, and winsor caps plus median imputation fit only on `snapshot_date < train_cutoff` (last two cohort months reserved for out-of-time evaluation).

Implementation: `utils/data_processing_{bronze,silver,gold}_table.py`, orchestrated by `main.py`. Further design detail: `bronze_layer.md`, `silver_layer.md`, `gold_layer.md`.

## Prerequisites

- [Docker Desktop](https://www.docker.com/products/docker-desktop/) (recommended)
- The four CSV files under `data/`

## Run the pipeline (Docker)

From the repository root:

```bash
docker compose build
docker compose up -d
docker compose exec pipeline python main.py
```

The project folder is bind-mounted to `/app`, so `datamart/` is written on your host.

**Runtime:** Roughly **20–40 minutes** for a full run (bronze + silver + gold over monthly snapshots from 2023-01 through 2025-11).

### What success looks like

Near the end of the log:

```text
[gold:feature] train_cutoff=2025-06-01 (winsor p99 + impute median fit on snapshot_date < 2025-06-01 only)
[gold:feature] reconcile customers=12500 attr=12500 fin=12500
```

Outputs:

- `datamart/gold/label_store/gold_label_store_*.parquet/` — 25 partitions (mob=6 cohort months, ~2023-07 through 2025-07)
- `datamart/gold/feature_store/gold_feature_store_*.parquet/` — same cohort months, 1:1 with labels

`[gold:feature]` partition lines may print out of chronological order; filenames are still dated correctly.

### Optional: sanity model (notebook)

Quick check that gold tables join and out-of-time performance is reasonable. Not required to run `main.py`.

1. Open **http://localhost:8888/lab** (JupyterLab in the container).
2. Install ML packages once:  
   `docker compose exec pipeline pip install scikit-learn xgboost`
3. Open `model_train.ipynb`, run a Spark session cell first, then run all cells.

Expect ROC-AUC on train, validation, and OOT splits (OOT uses the same `train_cutoff` as the feature store).

## Run without Docker (local Python)

Requires **Java 17+** and Python 3.10+.

```bash
python -m pip install -r requirements.txt
python main.py
```

On Windows without `HADOOP_HOME`, some silver/gold writes may use a pandas/parquet fallback; Docker matches the Linux Spark write path.

## Generated data

`datamart/` is produced by `main.py` and listed in `.gitignore`. Clone the repo, run the pipeline, and outputs appear locally.

## Stop the environment

```bash
docker compose down
```
